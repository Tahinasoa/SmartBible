# Malagasy Bible Project: Data Pipeline & Display App

This project was built to solve a real problem: converting a raw XML Bible into a fast, reliable display system for live church services. I have maintained and used this system in production for over 3 years.

## 🏗️ How it works

The project is divided into two parts:

1. **Data Pipeline (Python)**: Cleans the raw XML and moves it into a structured SQLite database.
2. **Display App (C++/Qt)**: A desktop software to search and show verses in full-screen with no lag.

------

## ⚙️ Part 1: Data Pipeline (The "ETL" process)

Raw XML files are often "dirty" (comments, repetitions, bad formatting). This Python tool fixes that before the data is used.

### What I solved:

- **Text Cleaning**: Used `BeautifulSoup` and `Regex` to remove translator notes (Gr, Heb, Na) while keeping a log of what was deleted.
- **Deduplication**: Wrote a script to find and remove repeating text at the end of chapters.
- **SQL Structuring**: Converted nested XML into a clean database with two tables: `books` and `verses`. This makes searching much faster than reading a text file.

**Tech used**: Python 3, BeautifulSoup4, Regex, SQLite3.

------

## 🖥️ Part 2: Display Application (C++)

I chose C++ and Qt for the display part because it needs to be very stable and fast during a live event.

- **Fast Search**: It talks directly to the SQLite database for instant results.
- **Proven Stability**: Used every week for 3 years without crashing.
- **Performance**: Very low memory usage compared to web-based tools.

------

## 🚀 How to Run

### 1. Data Pipeline

1. Run the ETL process:

   Bash

   ```
   python main.py
   ```

2. The optimized `baiboly.db` will be generated in the `output/` directory.

### 2. Build the SmartBible

1. Open the `.pro` file in **Qt Creator** and build using the Qt5 kit.
2. **Important**: Once the executable is built, manually copy the `baiboly.db` file (generated by the pipeline) into the same folder as the executable so the app can access the data.
